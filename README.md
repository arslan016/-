#Weather Diary (Дневник погоды)
Автор: Арслан Арсланов

Краткое описание программы
Weather Diary — это десктопное приложение для учёта и анализа погодных условий. Программа позволяет:

добавлять записи о погоде с указанием даты, температуры, описания и наличия осадков;

просматривать все сохранённые записи в виде таблицы;

фильтровать записи по дате и температуре;

сохранять данные в JSON‑файл и загружать их при следующем запуске;

удобно работать с данными через графический интерфейс.

Как запустить приложение
Требования:

Python 3.6 или выше;

стандартные библиотеки Python (tkinter, json, datetime) — не требуют отдельной установки.

Пошаговая инструкция:

Клонируйте репозиторий на свой компьютер:

bash
git clone <ссылка_на_репозиторий>
Перейдите в папку проекта:

bash
cd WeatherDiary
Запустите приложение:

bash
python src/weather_diary.py
Примеры использования
Пример 1. Добавление новой записи

Введите дату в формате ДД.ММ.ГГГГ (например, 15.04.2024).

Укажите температуру (например, +18 или -5).

Напишите описание погоды (например, «Солнечно, без осадков»).

Отметьте наличие осадков: «Да» или «Нет».

Нажмите кнопку «Добавить запись».

Новая запись появится в таблице внизу окна.

Пример 2. Фильтрация записей

По дате:

В поле «Фильтр по дате» введите нужную дату (например, 20.04.2024).

Нажмите «Применить фильтры».

В таблице останутся только записи за указанную дату.

По температуре:

В поле «Фильтр по температуре (>):» введите пороговое значение (например, 10).

Нажмите «Применить фильтры».

В таблице появятся записи с температурой выше указанного значения.

Пример 3. Сохранение и загрузка данных

Добавьте несколько записей о погоде.

Нажмите кнопку «Сохранить в JSON» — данные сохранятся в файл data/weather_records.json.

Закройте приложение.

Запустите его снова.

Нажмите «Загрузить из JSON» — все ранее сохранённые записи загрузятся в таблицу.

Возможные тесты для проверки работы
Тест	Действие	Ожидаемый результат
1	Попытка добавить запись с пустой датой	Появление сообщения об ошибке: «Неверный формат даты. Используйте ДД.ММ.ГГГГ.»
2	Ввод температуры в виде текста (например, «двадцать»)	Сообщение об ошибке: «Температура должна быть числом.»
3	Пустое поле «Описание погоды»	Сообщение об ошибке: «Описание погоды не может быть пустым.»
4	Добавление корректной записи	Запись появляется в таблице, поля ввода очищаются
5	Фильтрация по несуществующей дате	Таблица пуста (нет записей за указанную дату)
6	Сохранение данных	Файл data/weather_records.json создаётся/обновляется, появляется сообщение «Записи успешно сохранены в файл!»
7	Загрузка данных после перезапуска	Все сохранённые записи отображаются в таблице
Формат данных (JSON)
Данные сохраняются в файле data/weather_records.json в следующем формате:

json
[
  {
    "date": "15.04.2024",
    "temperature": 18.0,
    "description": "Солнечно, без осадков",
    "precipitation": "Нет"
  },
  {
    "date": "16.04.2024",
    "temperature": 12.5,
    "description": "Пасмурно, небольшой дождь",
    "precipitation": "Да"
  }
]
Примечание: файл weather_records.json не отслеживается в Git (добавлен в .gitignore), чтобы избежать конфликтов при совместной разработке. Вы можете создать пустой файл в папке data/ для удобства.
